# Frontend Structure

Canonical React/TypeScript project layout for the `webdev-go-react-psql` template.

## Directory layout

```text
frontend/
├── public/
├── src/
│   ├── api/
│   │   ├── client.ts              # axios/fetch client with auth + request ID interceptors
│   │   ├── generated.ts           # or any generated API client shim
│   │   └── hooks.ts               # react-query hooks wrapping endpoints
│   ├── components/
│   │   ├── ui/                    # buttons, inputs, dialogs (design-system atoms)
│   │   └── features/              # domain components (ProjectList, ChatMessageForm)
│   ├── hooks/
│   │   ├── useAuth.ts             # current user + auth state
│   │   ├── useCentrifuge.ts       # WS connection lifecycle
│   │   └── useWsEvent.ts          # subscribe to specific event types
│   ├── gen/
│   │   └── models.ts              # generated by gogen; READ-ONLY
│   ├── ws/
│   │   ├── centrifuge.ts          # client setup and token refresh
│   │   └── dispatch.ts            # type-safe event dispatch map
│   ├── routes/
│   │   └── ...                    # route definitions and route guards
│   └── main.tsx
├── tests/
│   └── ...                        # integration tests, MSW handlers
├── package.json
├── tsconfig.json
└── vite.config.ts
```

## Rules

1. **Generated types are the source of truth**. Import from `src/gen/models.ts`; do not duplicate shapes.
2. **Strict TypeScript**. `strict: true`, `noImplicitAny: true`.
3. **Components are data-bound, not data-owning**. API and WS logic live in hooks.
4. **Error states are first-class**. Every async flow has `loading`, `error`, and `idle`/`success` states.
5. **Auth guards at the route level**. Public routes are explicit; everything else requires a validated user.
6. **No secrets in the frontend bundle**. API base URL and public keys only; everything sensitive is server-side.
