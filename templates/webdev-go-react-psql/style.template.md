# Style Contract: <Project>

This is the stack-specific style contract. Generic rules live in `agent_bay/workflow/03-style.md`. Document any deviations and project-specific conventions here.

## 1. Error handling

- Go: handlers return `error`; no `panic` in request paths.
- Use `fmt.Errorf("<context>: %w", err)` at every boundary.
- TS: return `Result<T, ApiError>` or propagate errors to a boundary component.
- Map DB errors to domain errors in the service/repo, then to HTTP statuses in the handler.

## 2. Type safety

- `go 1.22+` with modules.
- `tsconfig.json` with `strict: true`, `noImplicitAny: true`, `noUncheckedIndexedAccess: true`.
- Generated types from sqlc, gogen, or Protobuf are the source of truth.
- No `any` except at the absolute edge of external payloads.

## 3. Testing

- Go: table-driven tests with `github.com/stretchr/testify/require`.
- Repository tests use a temporary Postgres database via `dockertest` or `testcontainers-go`.
- Handler tests call the real router with injected fakes.
- TS: `vitest` for logic and hooks; MSW for API mocking.

## 4. Authorization

- Extract `userID` from validated JWT in middleware.
- Every handler must check `authorize(ctx, actor, action, resource)`.
- Maintain a `docs/auth_matrix.md` if the rules become non-trivial.

## 5. Security rules

- Input validation with explicit schemas (`validator.v10` in Go, `zod` in TS).
- sqlc / parameterized queries only.
- CSP, CORS, rate limiting configured in `security.go` or middleware.
- Secrets in env only; never committed.

## 6. Logging & observability

- Use `slog` with JSON handler in Go.
- Attach request ID and user ID to context.
- Log every error once at the handler boundary.

## 7. Project-specific conventions

Package layout example for Go:

```text
cmd/<app>/main.go          # entrypoint, wiring
server/                    # HTTP server setup, middleware
handler/http/              # thin HTTP handlers
service/                   # business logic and authorization
model/                     # domain types shared by service + db
db/
  query/                   # sqlc generated code
  migration/               # up/down migrations
ws/                        # Centrifuge publishers
ws_api/                    # WS contract structs (Go) → gogen
```

Frontend layout:

```text
frontend/src/
  api/                     # generated + handwritten API clients
  components/              # UI components
  hooks/                   # data + WS hooks
  gen/                     # generated TS types
  ws/                      # Centrifuge subscription + dispatch
```

Generated code ownership:

- `db/query/*.go` — generated by sqlc, read-only.
- `ws_api/*.go` — hand-written Go contract structs; consumed by `gogen`.
- `frontend/src/gen/models.ts` — generated by `gogen`, read-only.
- `api/gen/` — generated from Protobuf, read-only.
